Digital Twin Logistics

QR-Based Digital Twin Logistics System

Period

Apr 2025 – Oct 2025

Type

Team project · Hanium competition

Stack

Python, C#, C++, ROS 2, MoveIt 2, YOLO, Unity

System

QR information was queued, conveyor arrival was detected by an ultrasonic sensor, and the robot arm executed destination-specific pick-and-place while Unity visualized system state.

Engineering Challenge

The main challenge was coordinating communication delay and execution order across perception, conveyor control, manipulation, and simulation. A UDP-based state exchange and explicit control sequence were used to synchronize the physical system and digital twin.

Robotic Manipulation

RGB-D Robot Arm Pick-and-Place

Period

Jul 2025 – Feb 2026

Type

Personal project · IRC Lab seminar

Stack

ROS 2, Python, RealSense D435, YOLOv11 Segmentation, tf2, MoveIt 2

Pipeline

RGB-D input → segmentation mask → depth-based 3D estimation → camera-to-robot transform → MoveIt 2 target pose and pick-and-place.

Engineering Challenge

The project moved from AprilTag-based estimation to markerless segmentation. Depth noise and coordinate-frame validation were handled through mask-based sampling, tf2, and RViz2 verification.

Mobile Robotics

Autonomous Following & Towing Robot

Period

Sep 2025 – Jun 2026

Role

System integration

Stack

ROS 2, RPLiDAR S2, ros2_control, SLAM, Nav2, PyQt5, RS485

System

The robot supports worker following, path recording, repeated autonomous navigation, cart towing, status monitoring, and fine manual control through a GUI.

Engineering Challenge

Functional code was reorganized into hardware, tracking, path, navigation, GUI, and mode-management layers. A staged SIGINT → SIGTERM → SIGKILL shutdown strategy reduced process conflicts during mode transitions.
